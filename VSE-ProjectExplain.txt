📁 src/

This is the main source folder of the Angular application.

Everything that runs in the browser lives here.

Rule:
UI, routing, services, guards, layouts → all go inside app.

1️⃣ Core Folder (Important for Interview)

To avoid duplicate logic and keep the app clean.

What it does

Protects routes

Why guard?

Security at routing level

Prevent URL hacking

Interview line:

What it does

Automatically attaches JWT token to every HTTP request

HOW FRONTEND & BACKEND CONNECT (VERY IMPORTANT)
🔁 COMPLETE FLOW
1. Angular Login → POST /auth/login
2. Backend returns JWT
3. Angular saves token in localStorage
4. Angular interceptor adds Authorization header
5. JwtAuthFilter validates token
6. Controller executes




Interview-ready explanation (short)

We implemented stateless authentication using JWT.
Login API generates a signed JWT which is stored in browser localStorage.
Angular HTTP interceptor automatically attaches the token to every API request.
On backend, a custom OncePerRequestFilter validates the JWT and sets authentication in Spring Security context.
Route guards on frontend and filter chains on backend together protect secured APIs

FLOW EXPLAINED (INTERVIEW ANSWER)

On the dashboard we display submissions using Angular Material table.
Each row has a delete button.
When clicked, Angular calls a service method which sends a DELETE request to Spring Boot.
The backend deletes the record using JPA repository.
After successful deletion, the dashboard refreshes the data.
JWT authentication secures the API via interceptor and security filter.
